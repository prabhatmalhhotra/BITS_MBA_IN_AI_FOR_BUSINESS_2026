Fundamentals of Hypothesis Testing

The scientific method in statistics: formulate a claim, collect evidence, and make a decision. This is how businesses prove (or disprove) assumptions with data.

T1 Chapter 9 (Levine) Comprehensive Exam ~55 min read
Real-World Problem

Tata Starbucks claims: "Our average service time is under 5 minutes."

A mystery shopper agency samples 40 Starbucks outlets and records a mean service time of 5.3 minutes with SD = 1.2 minutes. Is Starbucks' claim false, or is 5.3 just due to random sampling variation?

Hypothesis testing gives you a rigorous framework to answer: "Is the observed difference real or just noise?" with a quantifiable error rate.

1 The 7-Step Hypothesis Testing Framework Beginner

Step 1: State H₀ and H₁
H₀: status quo (=); H₁: research claim (≠, >, <)
▼
Step 2: Choose α (significance level)
Typically 0.05 (5%) or 0.01 (1%)
▼
Step 3: Choose test statistic
t, Z, χ², F depending on the problem
▼
Step 4: State decision rule
Reject H₀ if test stat falls in critical region
▼
Step 5: Collect data, compute statistic
Calculate t or Z from sample data
▼
Step 6: Compare (or compute p-value)
p < α ⇒ reject H₀
▼
Step 7: Business conclusion
Interpret in plain English for stakeholders

Type I (α) — False Alarm

  • Rejecting a true H₀
  • Controlled by choosing α
  • Example: Concluding a drug works when it doesn't

Type II (β) — Missed Detection

  • Failing to reject a false H₀
  • Decreases with larger n, larger effect, larger α
  • Example: Missing a real improvement in service time
Power = 1 - β    (probability of correctly detecting a real effect)

Power increases with: ↑ n, ↑ effect size, ↑ α
Daily Life Example

Fire alarm analogy: Type I error = alarm goes off when there is no fire (false alarm, annoying but safe). Type II error = no alarm when there IS a fire (missed detection, dangerous). You want both to be low, but Type II is usually more dangerous.

3 t-Test for the Mean (σ Unknown) Intermediate

t = (X̄ - μ₀) / (S / √n)     df = n - 1

Two-Tail Test (H₁: μ ≠ μ₀)

Reject H₀ if |t| > t(α/2, n-1) or equivalently if p-value < α.

One-Tail Tests

DirectionHypothesesReject H₀ if
Upper-tailH₀: μ ≤ μ₀; H₁: μ > μ₀t > t(α, n-1)
Lower-tailH₀: μ ≥ μ₀; H₁: μ < μ₀t < -t(α, n-1)

Worked Example (Starbucks)

H₀: μ ≤ 5 (claim is true), H₁: μ > 5 (service is slower). α = 0.05, n = 40, X̄ = 5.3, S = 1.2.

StepValue
t-statistic(5.3 - 5) / (1.2/√40) = 0.3 / 0.1897 = 1.582
t-critical (0.05, df=39)1.685
Decision1.582 < 1.685 ⇒ Fail to reject H₀
ConclusionNot enough evidence to say service time exceeds 5 min at 5% level

4 Z-Test for the Proportion Intermediate

Z = (p̂ - π₀) / √[π₀(1 - π₀) / n]

Note: under H₀, we use the hypothesised π₀ (not the sample p̂) in the standard error.

Trending Example (2024)

Swiggy Instamart claims >95% of orders delivered within 15 minutes. A consumer watchdog samples 300 orders and finds 278 delivered on time (p̂ = 92.7%). Z = (0.927 - 0.95) / √(0.95 × 0.05/300) = -0.023/0.01258 = -1.83. For a one-tail test (α = 0.05, z-critical = -1.645), |Z| > 1.645, so they reject the claim. Evidence suggests the true rate is below 95%.

Exam Tip: The comprehensive exam will almost certainly have one hypothesis test problem. Follow the 7-step framework religiously. State H₀/H₁, calculate the test statistic, compare to critical value OR compute p-value, and write a business conclusion. Common error: using the wrong formula for SE (use π₀ for proportion tests, use S for mean tests).

Session 9 — Key Takeaways

  • 7-step framework: State hypotheses → Choose α → Test stat → Decision rule → Compute → Compare → Conclude
  • H₀ = status quo (always contains =); H₁ = research claim
  • Type I (α) = false positive; Type II (β) = false negative; Power = 1 - β
  • t-test for mean: t = (X̄ - μ₀)/(S/√n), df = n-1
  • Z-test for proportion: Z = (p̂ - π₀)/√[π₀(1-π₀)/n]
  • p-value: probability of data as extreme as observed, assuming H₀ true; p < α ⇒ reject
  • One-tail: directional H₁ (> or <); Two-tail: non-directional (≠)
