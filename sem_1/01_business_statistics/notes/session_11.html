Chi-Square Tests

Testing relationships between categorical variables and whether observed data fits expected patterns.

T1 Chapter 11 (Levine)Comprehensive Exam~45 min read
Class Notes AI Notes
Real-World Problem

Airtel wants to know: Does customer churn depend on their plan type?

Airtel categorises customers by plan (Prepaid, Postpaid, Broadband) and status (Active, Churned). They have a 2000-customer contingency table. Is plan type associated with churn, or are they independent?

A chi-square test of independence answers exactly this: "Are two categorical variables associated?"

1 The Chi-Square Statistic Beginner

χ² = Σ (O - E)² / E

O = observed frequency, E = expected frequency

The larger χ² is, the more the observed data deviates from what we expected under H₀. It is always ≥ 0 and follows the chi-square distribution with the appropriate df.

2 Test for Difference in Two Proportions Intermediate

Equivalent to the Z-test for two proportions (Session 10) but expressed as chi-square. df = 1. The relationship is χ² = Z².

3 Test for Differences Among >2 Proportions Intermediate

df = c - 1   (c = number of groups)

Extension to c groups. H₀: π₁ = π₂ = ... = π₦. Compute expected counts assuming equal proportions, then calculate χ².

4 Test of Independence (Contingency Table) Intermediate

The most common chi-square test. Tests whether two categorical variables are independent.

Eᵢⱼ = (Rowᵢ total × Columnⱼ total) / Grand total

df = (r - 1)(c - 1)

Worked Example: Plan Type vs Churn

Plan \ StatusActiveChurnedTotal
Prepaid600200800
Postpaid45050500
Broadband550150700
Total16004002000

E(Prepaid, Active) = 800 × 1600 / 2000 = 640. Similarly compute all 6 expected values, then χ² = Σ(O-E)²/E. df = (3-1)(2-1) = 2.

Caveat: Each expected frequency Eᵢⱼ should be ≥ 5. If not, merge categories or use Fisher's exact test.

Trending Example (2024)

TRAI studying mobile plan preference vs age group: TRAI's 2024 annual report used chi-square tests of independence to demonstrate that age group (18-25, 26-40, 41-60, 60+) and plan preference (Data-heavy, Voice-heavy, Balanced) are significantly associated (p < 0.001). Younger users overwhelmingly prefer data-heavy plans.

5 Goodness-of-Fit Test Advanced

Tests whether observed data matches a hypothesised distribution (uniform, Poisson, normal, etc.).

df = k - 1 - (number of parameters estimated from data)

Example: A die is rolled 600 times. If fair, each face should appear 100 times. If observed counts are 90, 105, 95, 110, 100, 100, compute χ² = Σ(O-100)²/100 and compare to χ²-critical with df = 5.

Daily Life Example

Is a Ludo die fair? Roll it 300 times, record how often each face appears. Expected: 50 each. If one face shows up 80 times, the goodness-of-fit test tells you whether this deviation is beyond chance. That is the chi-square goodness-of-fit test.

Exam Tip: Know the difference in df calculation: Independence test: (r-1)(c-1). Goodness-of-fit: k-1 minus estimated parameters. Two proportions: df = 1. Also, always check Eᵢⱼ ≥ 5 and mention it in your answer.

Session 11 — Key Takeaways

  • χ² = Σ(O-E)²/E — always ≥ 0, right-tailed test
  • Two proportions: df = 1 (equivalent to Z²)
  • Multiple proportions: df = c - 1
  • Independence: E = row total × col total / grand total; df = (r-1)(c-1)
  • Goodness of fit: df = k - 1 - parameters estimated
  • All expected frequencies should be ≥ 5
  • Excel: =CHISQ.TEST(observed, expected) returns p-value directly
